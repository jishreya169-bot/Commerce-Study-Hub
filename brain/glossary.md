# Project Glossary

This document defines key business domain terms and concepts used across the Vidya Path system.

---

## 1. User Roles & Identity
* **Student**: An end-user enrolled in commerce classes. Can view lectures, download notes, complete mock tests, and submit doubts.
* **Teacher / Instructor**: An educator who publishes coursework, answers doubts, and evaluates student test profiles.
* **Admin**: An administrator who manages schedules, enrollments, reports system health, and reviews financial transactions.
* **Parent**: A profile linked to a Student to monitor academic progress and billing details.

---

## 2. Platform Core Entities
* **Landing Page**: Public introductory portal showing product metrics, hero descriptions, testimonials, and link pathways.
* **Course**: A subject package (e.g. "Class 12 - Accountancy") containing lectures, notes, and tests.
* **Lecture**: An educational video lesson (Recorded format).
* **Doubt**: A support ticket posted by a student query, which can be resolved by teachers or browsed via tab interfaces.
* **Notes / Materials**: Downloadable study materials, usually stored as PDFs or slides.
* **Mock Test**: Online assessment tool matching board exam structures.
* **Enrollment**: Record mapping a student to a specific Course stream.
* **Finance Module**: Administrative transaction list tracking fees collections, outstanding balances, and defaulters.
* **Academics Module**: Interface for configuring subject allocations and class schedules.
* **Communication Panel**: Admin tool for publishing notifications, newsletters, or critical alerts to users.
* **Vidya Path**: The brand name for this Commerce Study Hub.
