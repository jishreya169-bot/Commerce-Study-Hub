# Module Dependency Graph

This document maps package and file dependency paths across the pnpm monorepo.

---

## 1. Package Dependencies

```
        ┌─────────────────────────────────────────────────┐
        │                 [artifacts/mobile]              │
        │             (Expo React Native App)             │
        └──────┬──────────────────────┬─────────────┬─────┘
               │                      │             │
               ▼                      ▼             ▼
      [lib/api-client-react]     [lib/api-zod]   [Firebase Services]
     (Generated Query Hooks)    (Zod Schemas)   (Auth / Firestore)
               │                      │
               ▼                      ▼
         [lib/api-spec]         [artifacts/api-server]
      (OpenAPI Spec/YAML)         (Express TS Server)
                                      │
                                      ▼
                                  [lib/db]
                            (Drizzle ORM Schema)
                                      │
                                      ▼
                                 [PostgreSQL]
```

* **`artifacts/mobile`**: Relies on `@workspace/api-client-react` for REST queries, and references custom layout structures locally.
* **`lib/api-client-react`**: Code is auto-generated by Orval using specs from `lib/api-spec`.
* **`artifacts/api-server`**: Depends on database models exported from `lib/db`.
* **`lib/db`**: Standalone Drizzle schemas connected to the PostgreSQL database.

## 2. Mobile Routing Dependency Map
* **Root layout**: `app/_layout.tsx` (loads Poppins fonts, provides Auth, App, and Theme contexts, executes Redirect gates).
* **Landing Entrypoint**: `app/index.tsx` (unauthenticated Landing Page; routes to `app/login.tsx`).
* **Router stacks**:
  * `app/login.tsx` (Access gate screen; credentials validation).
  * `app/(tabs)` (Student sub-router layout).
    * Tab icons and indices depend on `courses.tsx`, `doubts.tsx`, `materials.tsx`, `notifications.tsx`, `profile.tsx`, `tests.tsx`.
  * `app/(teacher)` (Teacher sub-router layout).
    * Direct dependencies: `classes.tsx`, `doubts.tsx`, `notifications.tsx`, `profile.tsx`, `students.tsx`, `upload.tsx`.
  * `app/(admin)` (Admin dashboard sub-router layout).
    * Sub-pages: `academics.tsx`, `communication.tsx`, `courses.tsx`, `finance.tsx`, `notifications.tsx`, `profile.tsx`, `reports.tsx`, `students.tsx`, `teachers.tsx`.
